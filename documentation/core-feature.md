# resumelon Core Product Flow

## Product Philosophy

resumelon is not just a resume builder.

It is a recurring optimization tool designed to help users tailor their resume for every job they apply to.

The product revolves around:

- A **Primary Resume** (the foundation)
- **Optimized Resumes** (job-specific versions)

---

# 1. First-Time User Flow

## Step 1: Create Primary Resume

When a new user joins, their first goal is to create their Primary Resume.

They can:

- Create from scratch
- Upload an existing resume

The user goes through the step-based builder and AI assistance.

Once completed, this becomes their **Primary Resume**.

The Primary Resume:

- Is fully editable at any time
- Acts as the base for all future job optimizations
- Is not tied to any specific job

---

# 2. Dashboard Experience

After creating their Primary Resume, users land on the dashboard.

Primary action:

**Optimize For Job**

The dashboard shows:

- The Primary Resume
- All Optimized Resumes created from it

Each Optimized Resume is tied to a specific job.

---

# 3. Recurring Flow: Applying to a Job

This is the core loop of the product.

## Step 1: User Clicks “Optimize For Job”

They paste in:

- Job description
- Optional company details

They can choose:

- Analyze First
- Quick Optimize

---

## Step 2: (Optional) Job Analysis Screen

If they choose Analyze:

They see:

- Skills they already match
- Missing keywords
- Areas that could be strengthened
- Suggested improvements

They can:

- Accept or reject suggestions
- Adjust tone or emphasis

Then they generate the optimized resume.

---

## Step 3: Optimized Resume Created

The system creates a new resume derived from the Primary Resume.

This new resume:

- Is tied to that job
- Can be edited independently
- Does not change the Primary Resume

---

# 4. Editing an Optimized Resume

The user can:

- Open it
- Edit it directly
- Use AI tools to refine sections
- Regenerate content
- Export it

Each optimization can create a new version if needed.

---

# 5. Ongoing Usage Pattern

Over time, the user:

1. Maintains their Primary Resume
2. Creates Optimized Resumes for different jobs
3. Edits each version as needed
4. Exports and applies

The app becomes a tool they return to for every application.

---

# Core Mental Model

Primary Resume = Foundation  
Optimized Resume = Job-specific version

Users build once, optimize repeatedly.

---

# Chronological User Journey

1. Create Primary Resume
2. Save and refine it over time
3. Find a job
4. Paste job description
5. Optimize
6. Edit
7. Export
8. Repeat

---

# Product Goal

Make resume creation a one-time effort.

Make resume optimization a recurring habit.
